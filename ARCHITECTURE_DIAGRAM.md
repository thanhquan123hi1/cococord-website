## ğŸ” Key Design Decisions

1. **MongoDB for Messages**

   - Schemaless â†’ Flexible message structure
   - Horizontal scaling for high volume
   - Rich querying capabilities

2. **MySQL for Channels/Users**

   - Structured data with relationships
   - ACID transactions
   - Complex joins for permissions

3. **STOMP over WebSocket**

   - Standard protocol
   - Message routing built-in
   - Subscription management
   - SockJS fallback

4. **JWT for Auth**

   - Stateless authentication
   - Works with WebSocket
   - No session management needed

5. **Client-side Caching**

   - Reduce server load
   - Faster UI updates
   - Better UX

6. **Broadcast Pattern**
   - All subscribers get updates
   - Simple and reliable
   - No complex routing
